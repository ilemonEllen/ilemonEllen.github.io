政府的监控项目

这些是你很有必要知道的信息，虽然在生活中可能很少听到这些名词，但是它们确实真真实实的存在在我们的看不见的地方，与你我的生活息息相关。 防火长城 英文名字叫做 Great Firewall，简称：GFW，墙。 这个名字的是Geremie Barmé和Sang Ye发表于1997年6月1日的所写的文章《The Great Firewall of China》中提出的，也有人认为第一次使用 Great Firewall 的是 Charles R. Smith 在 2002年5月17日发布的文章《The Great Firewall of China》。 由北京邮电大学方滨兴在1998年领头设计打造，主要用于监控和过滤互联网国际出口内容的软硬件系统集合，分析和过滤中国境外网络的信息互相访问。是大陆地区的一种网络自我审查制度。而使用代理工具突破GFW封锁的行为叫做“翻墙，科学上网，自由上网”等。 GFW的功能十分强大，机制也十分的复杂，目前已知主要技术有以下这些： 1.域名解析服务缓存污染 例如：对Google的泛域名*.google.com进行了大面积的污染，所有以.google.com结尾的域名均遭到污染而解析错误不能正常访问。所以在大陆地区无法使用Google服务，包括Google的电子邮件。 2.IP地址或传输层端口封锁 3.针对TCP和UDP连接的封锁 4.TCP连接重置 5.中间人攻击 6.对电子通讯的拦截 7.网络攻击 其中网络攻击曾在2015年3月对GitHub进行为期五天的攻击，2021年3月份GitHub 443端口疑似被GFW进行DNS污染。 GFW在信息时代更像是一座网络牢笼，用以维稳。 金盾工程 全程叫做“全国公安工作信息化工程”，是由中华人民共和国北京政府提出的全国性网络安全基础建设工程，于2003年9月开工，2005年底完成。 全国公安工作信息化工程包括一个综合的、多层的、包括网络各个环节的封锁和监视系统，涉及技术（电信与网络服务提供商）、行政、公安、国安、宣传等很多部门的系统工程，是一个网络封锁与监视系统，可以用来看、听、及“思维”。 可以视作为一个综合的监控系统。 天网工程 也叫做天网系统，是中华人民共和国政府建成的世界上最大的视频监控系统，利用了人脸识别、大数据、人工智能等技术，是中国大陆大规模监控体系的一部分。 天网系统连接不同地方（例如：火车站、饭店、商场、剧院等公共场所，巴士、地铁、计程车等交通工具）的监控镜头，可在极短时间内识别大量民众身份。估计，到2020年中国大陆的社会信用体系基本建成并全面启用时，全国各地的公共监控摄像头数量将达到6.26亿个。 平安城市 平安城市为一综合性安防系统，主要资源来自政府。其透过三防系统(技防系统、物防系统、人防系统)以建设城市的平安和谐。为特大型且综合性强的管理系统，满足政府对于城市治安、交通管理等需求。 雪亮工程 雪亮工程是一个“民众性治安防范工程”，以县、乡、村三级综治中心为指挥平台，整体重点在于公共安全之视频监控。其利用三级综治中心使防范治安之措施拉近到民众自身身边，让民众们可以一起观看远距视频即时监控，共同参与治安防治，以实现“全覆盖、无死角”的治安防范。 绿坝-花季护航 绿坝·花季护航（简称绿坝或绿航）是中国大陆的一款客户端的网络过滤软件。政府称安装软件的目的在于保护未成年电脑用户，防止他们浏览色情网站等有害内容，但过滤的准确度过低。原定于2009年7月1日起，要求在中国大陆新生产和销售的电脑进行全面预装，后迫于舆论压力而表示无限期推迟该规定。 净网卫士 净网卫士，是中国大陆运营商中国电信推出的一款网络内容屏蔽软件，类似于2009年推出的“绿坝·花季护航”。两者的工作模式相似，都是通过部署在用户的操作系统中来达到屏蔽信息的目的。 由于该软件为木马软件，在苹果APP store无法上架，而安卓端，由于允许打开未知来源的安装包，因此无法遏制此情况。 有传闻说新疆等少数民族地区被推广强制安装，部分高校也出现类似的现象。 国家反诈中心 国家反诈中心是由公安部刑事侦查局开发的手机应用程序，发布于2021年3月。官方声称这是防骗软件，软件包含诈骗预警和上报功能，注册需要包含人脸信息。 它最厉害的地方莫过于能对诈骗行为进行提前预警，当您收到可疑诈骗分子来电，可疑诈骗分子发送的短信，或可疑短信内容网址，或安装可疑诈骗App应用时，它可以智能识别骗子身份并提前预警，大大降低您受骗的可能性。除此之外，它还可以：1.检测手机可疑应用 2.在线提交涉案线索 3.举报非法可疑电信诈骗行为 4.验证交易对方身份真实性 5.支付风险核验 6.收获专属防骗知识 这款应用在安卓系统上“请求高达29项权限”，甚至还需要用户上传敏感的人脸信息……令其相当怀疑这款“防诈骗”应用同时有监控民众隐私的目的。据报道，在广东省深圳市等地区实行强制安装此应用程序，据用户反映，存在监控VPN的现象。 google play store将其列为危险应用，未予上架。 数字人民币 数字人民币（E-CNY），又称数字货币电子支付（英语：Digital Currency Electronic Payment，缩写：DC/EP），是中国人民银行发行的法定数字货币。...

关于hugo博客的安排

回想 好久没有在hugo上更新了，这段时间里发生不少的事情。首先，我自己已经成功搭建部署了一个hexo博客对于我来说这无疑是自身的一种突破，之前尝试了很多次都是不清楚哪里的依赖出了问题，后来才转向搭建hugo的。其次，之前我的GitHub账号疑似被机器人监视了。 我注意到那个账号注册时间是七个小时之前，但是已经关注了200+人，没有任何的活跃记录，没有任何的star，所以我觉得很是可疑，因为我看到过消息说有些是机器人账号，用来监视一些不被当局所认可的项目。因此我决定暂时对于这个GitHub账户进行“冷藏”。我自己心里很清楚，我如果不那么做的话，我很可能会有别的麻烦。 hexo or hugo 最后的问题就是，我现在觉得hexo用来比hugo要来的顺手，原因我觉得无非以下几点： 1.hexo的插件更多，可支持和扩展的功能也就多一些。hugo相对较少，而且hugo还分为hugo和hugo_extended两个版本，前者对于自定义CSS是不支持的，后者才能。 2.hexo的动画过渡效果更好，主题也更加的美观。hugo的主题一般不自带过渡动画，而且主题大多个人都不怎么喜欢。 3.hexo写作时对于html.css等标签支持更加的友好就算是Markdown链接也支持在新标签页打开。hugo现在的Markdown引擎只支持在内部进行跳转。所以说单就一个Markdown链接跳转的问题来说，我还是更加青睐hexo。 4.hugo虽说现在在GitHub上面的star也不少了，但是对于一个新手来说或者是非编程人士来说，可参考的搭建教程资料还是比较少的。 5.对于hexo来说，博客的生成不需要配置什么环境变量，但是hugo在搭建过程中，如果你是下载的二进制文件的话，就要对二进制文件进行环境变量的配置。每一次hugo新版本的发布，如果你要更新的话就要更新二进制文件。所以，个人觉得对于使用Windows初次搭建的用户来说不算是特别友好。 6.hugo的命令实在是让我觉得又长又臭，就单说博客预览这一件事情，hexo只要输入hexo s就可以了，但是hugo就是要完完整整的输入hugo server -D。而且只要当你把文章的draft改为true时候，才能实现预览，改为false才能生成本地静态资源提交上去。真的让人感觉到有些心累。 计划 hexo主要用来记录学习类的，hugo主要用来纯文字书写，主要是生活类的。后面关于日常隐私保护类和技巧类的博客应该会接着写。但是教程类的估计就只会些hugo博客相关的了。

Windows Terminal配置

Windows Terminal](https://github.com/microsoft/terminal)是一款集成了Windows上的PowerShell、CMD以及Windows Linux子系统的终端。Windows自身的命令控制台过于简陋，所以我们不妨使用Windows Terminal来替代cmd和power shell，只需要简单的配置一些功能，就能实现很多常用的Linux操作命令。过程如下： Vim配置 Vim是一款优秀的Linux文本编辑器。我们在Windows下编辑一些一些配置文件的时候很有作用。 1.首先，我建议先去下载一个Vim,安装的时候注意路径中不要包含中文路径。 2.然后，配置环境变量。找到Vim.exe的二进制文件位置，例如我的Vim安装位置在C:\Program Files (x86)\Vim\vim82,然后到设置——系统——关于——高级系统设置——环境变量——path——编辑——新建。然后将Vim.exe的路径复制粘贴上去。（例如我的就是C:\Program Files (x86)\Vim\vim82）。然后重新打开Windows Terminal终端输入vim -h检验环境变量是否配置成功。 Git Bash配置 GitBash是一个用于MicrosoftWindows环境的应用程序，被集成在git下面。简明扼要的说一句就是能够实现很多Linux命令操作。更多资料——https://zh.wikipedia.org/zh-cn/Git 其实，我自己在网上看到很多的将Git Bash配置到Windows Terminal下都是通过修改Windows Terminal中的JSON文件来实现的。但是实际中，Windows Terminal已经精简了很多的冗长的代码，添加配置文件也变得更加的简单。 1.首先，你需要去下载一个Git 或者是Git for Windows,两者区别我自己目前使用的是Git。 2.打开Windows Terminal终端，进入设置——添加配置文件。填写名称，选择文件路径（找到自己git bash.exe的二进制文件路径）例如我的git 的bash.exe在D:\BigDataTools\Git\Git\bin下。你可以添加一个图标（当然也可以不要）。之后重启终端启用git bash输入git –version看看能否得到git版本，可以的话就代表成功了。 但是，完成相关的bash配置之后我们会发现，当我们使用命令浏览磁盘下具有中文的文件夹时会出现乱码现象。其实，这个很好解决，我们只需要在Git文件路径下找到bash.bashrc文件，例如我的路径D:\BigDataTools\Git\Git\etc。 然后打开文件进行编辑，在文件后面添加 export LANG="zh_CN.UTF-8" export LC_ALL="zh_CN.UTF-8" 之后，保存退出，重启终端，这个时候我们就会发现问题解决了。 其实将Git Bash集成到Windows Terminal中之后，完全也可以使用Vim，不需要下载添加VIM。当然，你甚至可以选择一个cmder直接下载使用，内置git bash，无需任何的配置，省时省力。 参考博客：https://blog.csdn.net/jun8148/article/details/97245249 Bash图标:
